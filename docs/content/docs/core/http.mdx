---
title: HTTP Server
description: The Fiber-backed HTTP server, middleware stack, and abstract router used across all modules.
---

The `http` package wraps [Fiber](https://gofiber.io/) behind a set of interfaces so that modules
register routes without depending on Fiber directly.

## Key Interfaces

| Type | Purpose |
|------|---------|
| `Engine` | Lifecycle interface: `RegisterRoutes`, `MountRoutes`, `Start`, `Shutdown` |
| `Router` | HTTP method registration: `Get`, `Post`, `Put`, `Patch`, `Delete` |
| `Context` | Request/response abstraction: headers, params, query, body, JSON response |
| `Handler` | `func(Context) error` — the handler signature used by all modules |

## Middleware

Every request passes through:

- **Ray-ID** — generates a unique request identifier and attaches it as a response header for
  distributed tracing correlation.
- **Zap access logger** — structured request log lines with method, path, status, and latency.
  Configurable skip paths (e.g. `/metrics`, `/status`).

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `HTTP_APP_NAME` | `mannaiah-http` | Fiber app name |
| `HTTP_PREFORK` | `false` | Enable multi-process prefork mode. Disable when using Prometheus (single-process metrics). |
| `HTTP_SERVER_HEADER` | `mannaiah` | Value of the `Server` response header |
| `HTTP_READ_TIMEOUT_MS` | `30000` | Read timeout in milliseconds |
| `HTTP_WRITE_TIMEOUT_MS` | `30000` | Write timeout in milliseconds |
| `HTTP_IDLE_TIMEOUT_MS` | `120000` | Keep-alive idle timeout in milliseconds |

The server address is resolved from `CORE_HOST` and `CORE_PORT` when no HTTP-specific host/port
is provided.
